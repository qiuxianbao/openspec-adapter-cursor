- **现有实现与风险**：[说明当前实现的逻辑及潜在风险]

## 3. 存储与数据流设计

> **AI 指令**: 任何存储模型(如 MySQL、Redis)的变更设计，必须包含预估数据量大小、索引设计依据以及冷热数据分离考量。

- **数据库/表结构变更**：[表名、字段、索引变更]
- **缓存策略**：[缓存结构、失效策略]

## 4. 面向失败设计 (Design for Failure)

> **AI 指令**: 如果涉及外部 RPC/HTTP 调用，必须显式说明熔断/降级/重试机制。写操作接口必须设计防重 token 或数据库唯一索引兜底策略。

- **降级策略 (Fallback)**：[核心依赖故障时的降级方案]
- **熔断机制 (Circuit Breaker)**：[哪些接口需要熔断，阈值是多少]
- **幂等性设计 (Idempotency)**：[重试场景下的幂等键设计]
- **重试与死信队列 (Retry & DLQ)**：[异步/重试机制设计]

# 约束与边界
- 必须提供 2-3 个方案并分析利弊，禁止在用户选定前开始实现。
- 架构复杂度必须匹配项目实际规模。
- 必须遵循开发规范 `.cursor/rules/coding.mdc`。
