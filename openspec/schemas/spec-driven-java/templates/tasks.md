## 任务清单

### 准则

- 不得偏离 specs 与 design，任务拆解需符合 MECE 原则（相互独立，完全穷尽）
- 拆分顺序：数据层 → 接口层 → 业务层 → 测试
- 每个 Task = 可独立提交的原子变更（约 3–5 个文件）
- 每个子任务 MUST 使用 `- [ ] X.Y` checkbox 格式（apply 阶段依赖此格式追踪进度）
- 每个子任务精确到文件路径和函数/类签名；层级标签从 `[前置条件|数据层|接口层|业务层|测试]` 中选取
- 不得增加 specs/design 未定义的功能
- 实现阶段对同一子任务的自动修复（lint/编译）最多 3 次，仍失败则暂停并上报

### 1. **Task 1**: [任务名]

- **目标**: 一句话描述
- [ ] 1.1 [前置条件] 具体任务描述（含路径/签名）
- [ ] 1.2 [数据层] 具体任务描述
- [ ] 1.3 [接口层] 具体任务描述
- **摘要：涉及文件**:
  - path/to/File.java — 新增/修改，做什么
